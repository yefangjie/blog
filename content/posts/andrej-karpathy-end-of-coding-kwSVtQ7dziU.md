## 概述

本期来自 No Priors Podcast 的视频中，前 Tesla AI 总监、OpenAI 创始成员 **Andrej Karpathy** 分享了他对编程未来的深刻见解。他认为传统意义上的"编程"正在终结，取而代之的是与 AI Agent 协作的新时代。

## 核心观点

### 1. 传统编程的终结

Karpathy 认为"编程"(coding)已不再是正确的动词。开发者的工作方式发生了根本转变：

- **过去**：逐行编写代码，计算机执行
- **现在**：向 AI Agent 表达意图，AI 编写并执行代码
- **未来**：管理和编排多个 AI Agent 协作

> "代码甚至不再是正确的动词了" — Andrej Karpathy

### 2. AI Agent 范式转变

对话强调了一个根本性转变。传统模式中，人类写代码、计算机执行。而在新范式下：

- 人类描述意图和目标
- AI Agent 理解、规划并执行
- 开发者专注于高层问题解决而非实现细节

### 3. "循环时代"(Loopy Era)

Karpathy 将当前阶段描述为"循环时代"：

- **无限可能性**：AI Agent 几乎可以完成任何可描述的任务
- **技能问题**：一切都成为"技能问题"——只要你能描述，AI 就能做到
- **新瓶颈**：瓶颈从编程能力转移到问题表述和意图传达能力

### 4. 自动研究与自主 Agent

讨论中提到的关键发展：

- **AutoResearch**：能够自主进行研究的 AI Agent
- **自我优化**：能够优化自身指令和行为的系统
- **多 Agent 协作**：在复杂任务上并行工作的多个 Agent

### 5. 对开发者的影响

Karpathy 描绘了开发者角色的演变：

| 传统角色 | 新角色 |
|---------|--------|
| 代码编写者 | 意图传达者 |
| 实现细节关注 | 系统设计关注 |
| 单一 Agent 使用 | 多 Agent 编排 |
| 手动编码 | 自然语言描述 |

> "我必须每天16小时向我的 Agent 表达我的意志" — Andrej Karpathy

## 结论

Karpathy 的愿景是：传统编程正在让位于 AI Agent 编排。未来不属于那些能写出最高效代码的人，而属于那些最能向 AI 系统表达意图、管理复杂多 Agent 工作流程的人。

这是一个根本性的范式转变，将重新定义软件开发、AI 应用和人与技术的关系。
